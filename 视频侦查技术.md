## 第二节基本概念

### 一、数字图像：p7

- #### 1. **数字图像**
  
  - **解释**：空间坐标和灰度均不连续的、用离散数字（一般用整数）表示的图像。一幅图像可定义为一个二维函数f(x,y)，当x,y和幅值f为有限的、离散的数值时，称该图像为数字图像。
  - **补充**：由按一定间隔排列的亮度不同的像点构成，形成像点的单位称“像素”，是组成图像的最小单位。
  
  #### 2. **像素（Pixel）**
  
  - **解释**：组成图像的最小单位，也是用点阵方式描述图像的最小单位。具有固定的形状，但无固定尺寸，是硬件和软件所能控制的最小单位。
  
  #### 3. **数字图像的尺寸**
  
  - **解释**：指图像总的像素数，即长与宽分别包含的像素数量（如“100×100像素”），通常用“百万像素”表示数码相机的分辨率。
  
  #### 4. **分辨率**
  
  - **解释**：单位长度内所含有的像素数量，单位为dpi（dot per inch，每英寸点数）或ppi（pixels per inch，每英寸像素数）。分辨率越高，图像越清晰，但文件体积越大。
  
  - 分类：
  
    - **图像分辨率**：每英寸图像含有的像素数，直接影响图像质量。
    - **设备分辨率**：输出设备（如打印机）每单位长度的点数，固定不可更改。
    - **输出分辨率**：激光打印机等设备输出图像时每英寸产生的点数。
    - **屏幕分辨率**：打印灰度图像或分色图像所用网屏上每英寸的行数。
  
  #### 5. **位分辨率（位深）**
  
  - **解释**：衡量每个像素存储的信息位元数，决定图像的每个像素中存放的颜色信息。例如，24位RGB图像表示每个像素的红、绿、蓝三原色各用8位存储，共24位深度。
  
  #### 6. **灰度直方图**
  
  - **解释**：反映一幅图像中各灰度级与对应像素出现频率之间的关系，以灰度级为横坐标，频率为纵坐标绘制的图形。
  - 性质：
    - 只能反映灰度分布，丢失像素位置信息。
    - 一幅图像对应唯一直方图，但不同图像可对应相同直方图。
    - 图像分区域的直方图之和为原图像的直方图。
  - **应用**：判断图像量化是否恰当、确定图像二值化阈值、计算图像信息量（熵）等。
  
  #### 7. **亮度**
  
  - **解释**：图像中明暗程度的平衡，决定明暗色调的强度。通过增加或减少所有像素的灰度值实现图像整体变亮或变暗，灰度值范围为[0,255]（0为黑，255为白）。
  
  #### 8. **对比度**
  
  - **解释**：从黑到白的渐变层次，比值越大，色彩表现越丰富，图像越清晰醒目；比值小则画面灰蒙蒙。定义为不同颜色之间的差异程度。
  
  #### 9. **色调、色相、饱和度**
  
  - **色调**：颜色的基本趋向（如红、黄、蓝），是彩色彼此区分的特性，由光的波长决定。
  - **色相**：色彩的具体颜色（如“红色”“蓝色”），调整色相即改变颜色种类。
  - **:white_check_mark:饱和度**：颜色的深度或纯度，表示色相中灰色成分所占比例，取值范围为0%（最低，灰色）～100%（最高，纯色）。
  
  #### 10. **图像模式**
  
  - 解释：用于表现颜色的数学算法，决定图像的显示和打印方式。常见模式包括：
    - **位图模式**：仅用黑白两色表示图像。
    - **灰度模式**：使用256级灰度（0～255）表现图像，过渡平滑细腻。
    - **RGB模式**：基于红（R）、绿（G）、蓝（B）三基色光混合原理，通过不同亮度值组合生成颜色，属于“加色法”。
    - **CMYK模式**：基于青（C）、洋红（M）、黄（Y）、黑（K）四色油墨混合原理，通过吸收光的不同光谱生成颜色，属于“减色法”，用于印刷。
    - **HSB模式**：基于人眼对色彩的感知，通过色相（H）、饱和度（S）、亮度（B）描述颜色。
  
  #### 11. **图像文件格式**
  
  - 常见格式：
    - **BMP**：Windows标准点阵图像格式，支持多种色彩模式，不支持Alpha通道。
    - **GIF**：压缩率高，仅支持8位图像（256色），适用于网络传输。
    - **JPEG**：高压缩率，有损压缩，适用于图像预览和网络传输，易丢失细节。
    - **TIFF**：支持多种色彩模式和Alpha通道，兼容性强，常用于印刷和专业图像处理。
    - **PNG**：支持透明背景和无损压缩，适用于网络图像，支持24位真彩色。
  
  #### 12. **灰度直方图的统计参数**
  
  - **灰度平均值**：反映图像的整体明暗程度，计算公式为所有像素灰度值的算术平均。
  - **方差**：反映图像灰度值的离散程度，方差越大，图像对比度越高。
  - **标准偏差**：方差的平方根，同样用于描述灰度分布的离散程度。

### 二、视频：p14

- #### 1. **视频（Video）**

  - **解释**：将一系列静态影像以电信号方式捕捉、记录、处理、储存、传送与重现的技术。当连续图像变化每秒超过24帧（frame）时，根据视觉暂留原理，人眼会将其感知为平滑连续的动态画面。
  - **分类**：按内容变化性质可分为静止图像和活动图像（序列图像）；按色调可分为黑白视频和彩色视频。

  #### 2. **帧（Frame）**

  - **解释**：视频中的单幅静态图像，是视频信号的基础。视频通过连续播放不同内容的帧实现动态效果，常见帧率为25帧/秒（PAL制）或30帧/秒（NTSC制）。

  #### 3. **视频图像编码标准**

  - **定义**：由国际标准化组织（ISO）和国际电信联盟（ITU）制定的视频压缩编码协议，用于实现视频的高效存储与传输。
  - 主要标准：
    - **H.261**：用于视频会议，支持P×64kbps码率（P=1~30），处理CIF和QCIF格式图像。
    - **MPEG-1**：针对1.5Mbps以下数据传输率，用于VCD制作，分辨率为352×288（PAL）或352×240（NTSC）。
    - **MPEG-2**：支持更高码率（4~15Mbps），用于DVD和HDTV，兼容隔行扫描电视编码。
    - **H.263**：低码率（<64kbps）视频编码，适用于PSTN和移动通信，支持多种图像格式（如SQCIF、QCIF、CIF等）。
    - **MPEG-4**：基于对象的低码率编码，支持交互性和多媒体集成，可用于网络流媒体和移动视频。

  #### 4. **视频格式**

  - 分类：按应用场景分为本地影像视频和网络流媒体视频，常见格式包括：
    - **MPEG系列**：MPEG-1（.mpg/.dat）、MPEG-2（.vob）、MPEG-4（.mp4），支持不同压缩率和画质。
    - **DIVX**：基于MPEG-4改进的高压缩比格式，可将DVD画质视频压缩至CD容量。
    - **AVI**：音频视频交错格式，支持多种压缩算法，兼容性强但文件体积大。
    - **REAL VIDEO（RM/RAM）**：专为视频流设计，适用于低带宽网络（如56K Modem）。
    - **ASF**：微软推出的流式格式，采用MPEG-4压缩，平衡压缩率与画质。

### 三、音频：p19

- #### 1. **音频（Audio）**

  - **解释**：由监控视频采集卡附带的音频采集系统记录的监控现场环境声音，通过回放可再现现场声音，在案件中可用于辨别嫌疑人说话内容、认定嫌疑人身份。

  #### 2. **信噪比（Signal-to-Noise Ratio）**

  - **解释**：衡量录音器材质量的重要指标，指电子录音设备或系统中信号与噪声的比例。信号是需处理的电子信号，噪声是设备产生的无规则额外信号，信噪比越高，音频质量越好。

  #### 3. **动态范围（Dynamic Range）**

  - **解释**：声音信号最强部分与最微弱部分之间的电平差，反映录音器材对强弱信号的兼顾处理能力。动态范围越大，声音层次感越丰富，一般性能较好的音响系统动态范围在100dB以上。

  #### 4. **频率范围（Frequency Range）**

  - **解释**：音响系统能够回放的最低有效频率与最高有效频率之间的范围。自然界声音频率丰富，人耳听觉频率范围为20Hz~20kHz，音频信号频率范围越宽，包含的音频分量越丰富，音质越好。

  #### 5. **视频监控中音频的特点**

  - **背景噪声大**：监控摄像头多安装于公共场所，录制的音频信号易受环境杂音干扰，背景噪声较高。
  - **高频信号较弱**：摄像头通常安装位置较高，距离声源较远，导致录制的音频信号中高频成分衰减明显，清晰度较低。



## 第三节视频监控系统

**视频监控系统的种类与组成?**

- 模拟视频监控系统：以模拟摄像机、同轴电缆、模拟矩阵、磁带录像机为核心。20页

- 数字视频监控系统：采用PC式硬盘录像机（PC-DVR）实现模拟信号数字化和本地存储。20页

- 网络视频监控系统：通过嵌入式硬盘录像机（E-DVR）或视频服务器（DVS）接入IP网络，支持远程监控和多系统融合。20页

- 高清视频监控系统：采用网络高清摄像机（IPC），实现全IP接入和高清化、智能化应用。

- **组成**(p25)：信号采集设备（摄像机）、传输设备（光纤、网线）、存储设备（DVR、NVR、NAS、SAN）、显示设备（监视器、大屏系统）、控制设备（矩阵、解码器、数字高清矩阵）及管理设备（客户端、平台软件）。25页

**网络视频监控系统的原理与操控**

- **原理**：基于IP网络传输压缩后的视音频信号，前端设备（模拟摄像机+编码器或网络摄像机）采集信号并编码，通过以太网传输至后端，由NVR集中存储、管理和转发，支持远程访问和多系统联动。22页
- 操控要点：
  - 设备IP配置与网络调试，确保前端与后端通信畅通。
  - 视频流转发与存储策略设置，平衡带宽与存储资源。
  - 多系统融合（如与门禁、报警系统联动）的协议对接。

  

## 第四节视频侦查及应用

#### **一、视频侦查概述**

1. **定义**
   - 视频侦查技术是依照法定程序，以预防、控制、揭露和证实犯罪为目的，运用视频监控技术、计算机技术及其他信息化技术，分析、处理和检验与犯罪有关视频资料的专门活动。
   - **核心要素**：主体（侦查机关及人员）、客体（与犯罪有关的人、车、物品等）、方法（视频监控技术、计算机图像技术等）、目的（预防、控制、揭露、证实犯罪）。
2. **技术原理**
   - **同一认定与种属认定**：通过视频图像对特定客体（如嫌疑人、车辆）的特征（衣着、身形、步态等）进行同一性或种类属性判断。
   - **时空关系分析**：利用多监控点记录的时空信息，还原犯罪行为的时间序列（如案前踩点、实施犯罪、逃离路线）和空间轨迹。
3. **技术特点**
   - **准确性与直观性**：视频资料客观记录案件过程，可直接再现犯罪细节。
   - **物质依赖性**：依赖存储介质（如硬盘、服务器）和设备（如解码器）。
   - **时效性与连续性**：视频内容需及时采集（避免被覆盖），且能连续记录事件发展过程。
   - **易更改性**：可能被删除、编辑或伪造，需进行真实性检验。
   - **信息关联性**：视频中的人、车、物信息可与其他侦查手段（如卡口数据、通信记录）联动。

#### **二、视频侦查技术的作用**

1. **奠定打击犯罪基础**：视频监控系统普及为侦查提供关键线索，社会监控资源拓展覆盖范围。
2. **刻画犯罪嫌疑人**：通过视频分析嫌疑人的体貌特征、行为习惯、作案工具及逃跑路线。
3. **重建犯罪现场**：利用视频还原案件发生过程，验证证人证言和现场勘查结论。
4. **串并案件**：通过嫌疑人行为模式、工具特征等关联多起案件。
5. **震慑犯罪**：监控系统的存在降低犯罪发生率，实时监控可及时制止现行犯罪。
6. **辅助审讯与诉讼**：视频证据可突破嫌疑人心理防线，作为法庭呈堂证供。

#### **三、视频侦查技术的意义**

1. **推动城市信息化建设**：整合治安、交通等监控资源，服务社会管理与公共安全。
2. 革新警务工作模式：
   - **侦查领域**：开创“从像到人”“从像到案”等新型侦查模式。
   - **治安防控**：实现视频巡逻、智能预警，构建立体防控网络。
   - **交通管理**：通过卡口监控、违章抓拍提升执法效率。
3. **应对反恐处突**：全时段、全方位监控为快速响应和证据固定提供支撑（如伦敦地铁爆炸案侦破）。
4. **建设服务型公安**：优化资源共享，提升警务效率，服务群众需求（如寻人、追赃）。

#### **四、视频侦查组织机构**

1. **工作主体**：由专门视频侦查队伍负责，包括视频研判人员、涉案视频提取人员、实时巡查人员等。
2. 队伍建设：
   - **人员素质**：需具备计算机知识、数字影像处理能力、刑侦业务知识及沟通协调能力。
   - **职责分工**：省级负责统筹规划与技术支持，市级负责案件协调与培训，区县级负责具体案件侦查与设备管理。
3. **基础设施**：配备高性能计算机、移动存储设备、专业图像处理软件及视频分析平台。

#### **五、国内外发展应用状况**

1. 发展阶段：
   - **模拟监控**（20世纪70年代）：依赖磁带录像机，存储与传输受限。
   - **数字监控**（90年代末）：采用硬盘录像机（DVR）实现数字化存储。
   - **网络监控**（2005年起）：通过IP网络实现远程传输与集中管理。
   - **高清监控**（近年）：提升图像分辨率，支持智能分析（如人脸识别、行为预警）。
2. **国际应用**：英国伦敦通过密集监控系统快速侦破恐怖案件，美国建立法庭视频分析实验室。



# 第二章视频现场勘查
## 第一节视频现场勘查

### 一、视频勘查概述

#### **一、视频现场勘查的概念**

1. **定义**
   视频现场勘查是通过对犯罪现场及相关区域内视频监控系统、网络及其监视范围的调查访问，发现与收集视频监控记录的犯罪信息，以揭露犯罪行为的专门活动。其核心要素包括：
   - **主体**：视频监控系统（物质设备）。
   - **空间范围**：监控系统监视控制的场所及周边环境。
   - **实质内容**：与犯罪有关的时间、地点及人、事、物的视（听）信息。
2. **工作内容**
   - 调查监控前端（探头）的安装位置、方向、角度及监控范围、盲区。
   - 了解监控系统的工作原理、存储介质、播放软件等，评估其使用价值。
   - 检索、查看、采集与犯罪相关的视频信息资料。

#### **二、视频现场勘查与犯罪现场勘查的关系**

1. **包含关系**：视频现场勘查是犯罪现场勘查的重要组成部分，犯罪现场勘查包含视频现场勘查。

2. 拓展与延伸

   ：

   - **时间范畴**：突破犯罪行为实施时间限制，涵盖案前、案中、案后全周期。
   - **空间范畴**：不局限于犯罪行为实施地点，可跨区域、跨省市延伸。
   - **方法创新**：可通过网络远程勘查，突破实地勘查的空间限制。



### 二、视频勘查的内容

#### **三、视频现场的分类**

| **分类标准** | **类型**     | **核心特征**                                         |
| ------------ | ------------ | ---------------------------------------------------- |
| **记录场所** | 中心视频现场 | 犯罪行为发生场所的监控系统，直接记录犯罪过程。       |
|              | 外围视频现场 | 周边区域监控，辅助获取嫌疑人轨迹、活动规律等线索。   |
| **监控方式** | 固定视频现场 | 探头位置、方向固定，记录范围稳定。                   |
|              | 移动视频现场 | 监控设备可移动（如车载监控），记录内容具有随机性。   |
| **可视性**   | 可视现场     | 监控系统能有效记录清晰信息的区域。                   |
|              | 盲区现场     | 因遮挡、光照或未安装监控导致无法记录有效信息的区域。 |
| **操作功能** | 被动视频现场 | 按预设程序自动记录，无人工干预。                     |
|              | 主动视频现场 | 智能触发或人工控制记录（如红外感应、跟踪拍摄）。     |

#### :white_check_mark:**四、视频勘查的内容**

1. **划定区域和路线，踏勘监控点**
   - 分析监控点的地理位置、道路环境、光照条件，确定监控可视范围及盲区。
   - 评估监控对现场的封锁控制能力，预判嫌疑人可能出现的方向、时间段。
2. **制作视频现场分布图**
   - 标注监控探头位置、方向、范围及所属单位，明确监控之间的并列、交叉、连续关系。
   - 记录嫌疑人在各监控点的出现时间、移动速度及轨迹，辅助分析活动规律。
3. **访问监控系统**
   - **系统运行情况**：检查设备是否正常工作，校对系统时间误差。
   - **存储与播放**：了解存储介质类型、压缩方式、保存周期及播放软件兼容性。
   - **信息检索与采集**：结合案情检索目标时间段的视频，提取关键画面和片段。
4. **视频资料检索与查看**
   - **检索**：通过人工浏览或软件辅助，筛选与案件相关的视频片段。
   - **查看**：调节播放速度、亮度、对比度等参数，辨识嫌疑人特征、行为细节及环境信息。

### 三、视频现场勘查的任务和方法

#### **五、视频现场勘查的任务**

1. **查明犯罪现场情况**：通过视频信息佐证案件性质、案发过程，验证现场访问与勘验结论。
2. **发现和收集证据**：提取视频资料作为视听证据，固定犯罪嫌疑人的体貌特征、作案工具等。
3. **分析案情**：通过视频轨迹判断嫌疑人来去路线，确定侦查方向和范围。
4. **拓展侦查手段**：关联技侦、网侦等信息，拓宽线索渠道（如通过视频中手机使用记录锁定嫌疑人）。

- 设备因素
- :white_check_mark:环境因素
  - 1.低亮度图像（见图3-17）。在夜间，监控系统采集视频图像的过程没有补光或者补光不足，容易造成图像的曝光不足，无法对目标物进行辨别。
  - 2.低对比度图像（见图3-18）。由于监控摄像头的质量不高，或者天气环境等因素影响，形成图像的反差不强，画面中目标物与背景不能辨识。
  - 3.低宽容度图像。由于没有考虑监控系统的光线影响，没有采用宽动态的监控设备，造成记录的主要目标的亮度层次不清，无法对目标物进行辨别。
  - 4.荧光图像。在没有充分考虑环境光线影响，摄像头安装位置和角度不当，受多种光线干扰，特别是高亮光线或逆光干扰的情况下，容易造成图像的曝光过度，也无法对目标物进行辨别。
  - 5.褪色图像。由于监控系统建设过程中没有补光条件或补光不足而采用了红外摄像头，在红外监控下，目标灰度层次与真实场景不同，且有戴眼镜的假象，无法对目标物进行辨别。
  - 6.遮挡图像。由于监控系统维护不到位，摄像头长期被灰尘、周围树木、建筑物、电杆电线等遮挡，易使画面中的目标物被遮挡，影响对目标的辨别。

- 空间域平滑滤波处理
  - （1）邻域均值法。邻域均值法是将一个像元及其邻域中所有像元的平均值赋给输出图像中相应的像元，从而达到平滑的目的。
  - （2）加权平均法。加权平均法既能有效地滤除灰度图像中的噪音，又能很好地保护图像的边缘和细节，弥补了常用空域平滑滤波算法不能兼顾去噪和保持图像细节的不足。
  - :white_check_mark:（3）中值滤波法。中值滤波就是将邻域内所有像元值从小到大排序，取中间值作为中间像元的输出值。它是一种非线性滤波。在消除噪音的同时，能保持图像中的细节部分，防止边缘模糊。

- 带阻滤波器：能通过大多数频率分量、但将某些范围内的频率分量衰减到极低水平。
